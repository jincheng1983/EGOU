## 一、项目定位与目标

### 1.1 项目定位

EGOU 是一个以 **Go 语言为 runtime**、以 **中文为语法呈现** 的集成开发环境。借鉴易语言的交互范式（程序集、函数、表格+代码编辑器、窗口设计器、支持库面板），底层不依赖易语言私有编译器和二进制格式，而是将中文源码（.eg）转译为标准 Go 代码后调用官方 Go 工具链编译运行。

### 1.2 第八版的核心目标

1. **照搬第七版（EGOU）的成熟实现**：转译器、编译运行、窗口设计器、AI 编排、扩展包、主题系统等已验证可用的代码原样迁移。
2. **重构目录架构**：第七版根目录堆放 main.go / ide_service.go（1688+行/44 方法）/ examples.go 的问题彻底解决，采用标准 Go 项目布局。
3. **拆分巨型文件**：将 ide_service.go 按职责拆分到 internal/app/ 下多个文件，单一职责清晰。
4. **修复遗留问题**：CSS 变量缺失、硬编码颜色、AI 前端未接入、PropertiesPanel 占位等。
5. **完善未竟功能**：调试器、项目记忆、AST 编译器前端、i18n 等。
6. **一次性开发到位**：本版本是最终版，架构设计要前瞻、可扩展。

### 1.3 非目标

- 不重写已经成熟的转译器、AI 编排、运行时模板等核心逻辑。
- 不做向后兼容冗余代码（clean over compatible）。
- 不做一次性数据迁移功能。

### 3.1 架构设计要点

1. **入口标准化**：`cmd/egou/main.go` 是标准 Go 项目布局，便于未来扩展多入口（如 `cmd/egctl/` 管理工具）。
2. **IDEService 拆分**：第七版 1688 行单文件拆为 8 个职责文件，每个文件 200-300 行，可读性、可维护性大幅提升。
3. **字体内置**：根目录 `fonts/egou.ttf` 通过 `go:embed` 嵌入 IDE，作为默认字体。
4. **图标位置**：根目录 `appicon.png` / `icon.ico` 保持原位（Wails 约定）。
5. **internal/ 单一职责**：移除第七版 `core/`（与 `types/` 重复）和 `eco/`（与 `libs` 边界模糊）。
6. **文档集中**：所有文档在 `docs/` 下，规约在 `.trae/rules/`。

### 3.2 模块依赖关系

```
cmd/egou/main.go
    ├── constructs → internal/app.IDEService
    ├── embeds → frontend/dist
    └── registers → Wails application

internal/app/*
    ├── calls → internal/runner
    ├── calls → internal/transpiler
    ├── calls → internal/ai
    └── embeds → internal/examples

internal/runner
    ├── calls → internal/transpiler
    ├── aliases → internal/types
    └── embeds → runtime/

internal/transpiler
    └── imports → internal/types

internal/ai
    └── imports → internal/types

frontend/src
    └── wails_binding → internal/app.IDEService
```

依赖无环（`internal/types` 打破 runner ↔ app 循环依赖）。

## 六、UI 布局规约（沿用第七版，已验证）

### 6.1 三栏布局（强制）

```
┌─────────────────────────────────────────────┐
│                  TitleBar                    │
├──────┬──────────────────────────────┬───────┤
│ Left │         Editor Stack         │ Right │
│ 240px│         flex:1               │ 280px │
│      │                              │       │
│ 4 标签│                              │大纲/属性│
│ 文件  │                              │       │
│ 项目  │                              │       │
│ 支持  │                              │       │
│ AI   │                              │       │
├──────┴──────────────────────────────┴───────┤
│                 Output Panel                 │
└─────────────────────────────────────────────┘
```

- 左侧默认 240px（可拖拽调整，持久化 localStorage），4 标签：文件 / 项目 / 支持 / AI
- 中间 flex:1，文件标签页 + 编辑器/设计器
- 右侧默认 280px（可拖拽调整）：代码视图显示代码大纲，设计视图显示属性面板
- 底部输出面板（高度可拖拽调整），5 标签：输出 / 错误 / 提示 / 书签 / 历史
- 两侧 v-if 隐藏时其他列零影响（避免 4 栏布局失败）

### 6.2 标题栏规约

- "编译运行" 仅图标，无文字，透明背景 + 强调色图标，无边框，hover 效果与其他按钮一致
- "保存" 和 "另存为" 双按钮
- 扳手图标下拉菜单：仅"生成可执行文件 (Windows/Linux/macOS)"和"编译选项..."
- 左侧播放按钮：自动识别当前平台，触发编译运行
- 图标按钮用原生 `<button>`（不用 n-button，避免 Naive UI 内部样式干扰）

### 6.3 左侧边栏

- 禁止折叠功能（避免无法再展开）
- 4 标签固定顺序：文件 / 项目 / 支持 / AI
- 图标按钮用原生 `<button>`

### 6.4 主题规约

- 单一数据源：themes.js（4 套主题 × 39 变量，含圆角 5 档）
- 4 套主题：深色（dark）/ 浅色（light）/ 海蓝（ocean）/ 朝阳（sunrise）

## 七、编译运行规约

### 7.1 编译流程

```
用户保存 .eg
    ↓
IDE 调用后端转译器
    ↓
合并 .elib 扩展包源码（全局 libs/ + 项目 libs/）
    ↓
解析 .eg 生成 AST → 转译为标准 .go 文件
    ↓
处理 @嵌入 块
    ↓
写入 build/gen/ 目录
    ↓
复制运行时模板到临时目录
    ↓
写入用户代码 + 资源嵌入 + 前端资源
    ↓
go mod tidy
    ↓
go build（debug/release 双模式）
    ↓
运行 / 复制到产物目录
```

### 7.2 错误处理

- 结构化解析：`^(.+?):(\d+):(\d+):\s*(.+)$` 匹配 `file:line:col: message`
- 中文翻译：`undefined:` → `未定义:`，`cannot use` → `无法使用` 等 20 条规则
- 前端跳转：错误条目可点击跳转到对应文件行

### 7.3 版本号

- debug 模式：`egruntime.exe`（固定文件名）
- release 模式：`egruntime-v1.0.1-release.exe`（含版本号）
- 自动自增：release 构建后 patch+1，写回 `project.eg.json`（仅 release 模式）

### 7.4 产物

- 输出目录：`project.eg.json` 的 `output` 字段（默认 `bin`）
- 原生库：复制 native/ 目录到产物目录
- 打开文件夹：构建完成后自动调用 OpenInExplorer

## 十一、开发流程规约

- 严禁 Options API

- Excel 导入导出：优先使用 Excel
- 双加密：传输层 + 应用层（首次登录密码修改）

- 查询结果：不返回 Base64 编码
